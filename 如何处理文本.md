我们用一个 NLP 小任务贯穿：输入一句评论，判断是正面还是负面。 例如：
"这个电影很好看" → 正面
"这个电影太差了" → 负面

# 1 CNN如何处理文本
  CNN 原本常用于图像，但也可以处理文本。它处理文本的核心方式是：把文本看成一个“向量序列”，然后用卷积核扫描局部片段。
  具体流程可以表示为：
  输入文本矩阵 X ∈ R^(n × d)

→ 卷积核 W 在 X 上滑动
→ 每个位置计算匹配分数
→ 得到 feature map
→ ReLU 保留有效激活
→ Max Pooling 取最强激活
→ 多个卷积核得到多个特征
→ 拼成 sentence vector
→ 分类器输出类别概率

## 1.1 输入文本矩阵
  这个电影很好看 →  这 / 个 / 电 / 影 / 很 / 好 / 看
  再把每个token 变成 embedding 向量：
  这 → [0.12, -0.33, 0.45]
  个 → [0.08,  0.21, -0.19]
  电 → [0.44, -0.02, 0.17]
  影 → [0.31,  0.11, 0.29]
  很 → [0.22, -0.41, 0.36]
  好 → [0.75,  0.62, -0.10]
  看 → [0.69,  0.58, -0.08]
  整体可以看成一个矩阵：X ∈ R^(n × d)

## 1.2 每个位置计算匹配分数
  CNN它会拿一个固定大小的窗口，在矩阵 X 上滑动。
  假设某个卷积核参数为：

## 1.4 ReLU 保留有效激活
  接着经过激活函数 当前所有结果都是正数，因此结果不变：
      [0.354,0.126,0.279,0.122,0.687,0.516]

## 1.5 Max Pooling 取最强激活，，多个卷积核得到多个特征，拼成 sentence vector
  经过卷积和 ReLU 后，每个卷积核都会得到一串激活值：有 100 个卷积核。每个卷积核经过 max pooling 后得到 1 个数

## 1.6 分类器输出类别概率

  分类器拿到这个句子向量，判断类别。

  情感分类：

  sentence_vector
  → 分类器
  → [负面分数, 正面分数]

## 1.7 CNN的限制
  CNN它太依赖局部窗口。如果一个任务需要理解很远距离的信息关系，CNN 就不够自然。
  例如 
  前面大量铺垫，最后一句反转
  跨句指代
  复杂因果关系
  长距离否定
  多轮对话上下文
  因为它的核心机制是：局部扫描 + 强特征提取
